# 📋 Changelog v2.0 - Major Refactor

## 🚀 Что было добавлено

### 🎯 Ключевые фичи

#### 1. **Пошаговая воронка (Step-by-Step Funnel)**
- Пользователь больше не получает всю инструкцию сразу
- Каждый этап - отдельное сообщение с кнопкой
- 5 четких шагов:
  1. 📦 Скриншот товара в корзине
  2. 💳 Скриншот покупки
  3. 📦 Скриншот получения товара (ПВЗ)
  4. ⭐ Скриншот отзыва
  5. 💰 Реквизиты для выплаты

#### 2. **Telegram Web App (Админ-панель)**
- **Удобный веб-интерфейс** внутри Telegram
- **2 вкладки:**
  - 🛍️ **Товары**: Управление названием и инструкцией
  - 📝 **Тексты**: Редактирование всех сообщений бота
- **Проверка безопасности**: Только админы из списка
- **Адаптивный дизайн** под темную/светлую тему Telegram

#### 3. **Упрощенная модель товаров**
- **Было:** Название + Ссылка + Кэшбэк + Инструкция
- **Стало:** Название + Инструкция
- Вся информация (ссылка, сумма, условия) в одном тексте
- Админ сам пишет как удобно

#### 4. **Модерация заказов с подтверждением**
- Админу приходят **все 4 скриншота** + реквизиты
- Кнопки: ✅ Подтвердить / ❌ Отклонить
- **При подтверждении:**
  - Админ отправляет скрин перевода
  - Пользователь получает уведомление
  - **Только сейчас** данные записываются в Google Sheets (Лист1)
- **При отклонении:**
  - Админ указывает причину
  - Пользователь получает причину отказа

#### 5. **Настраиваемые тексты**
- Все тексты бота теперь в базе данных
- Редактируются через Web App
- Ключи конфигурации:
  - `welcome_message` - Приветствие
  - `products_select_text` - Текст выбора товара
  - `step_1_message` - Шаг 1 (корзина)
  - `step_2_message` - Шаг 2 (покупка)
  - `step_3_message` - Шаг 3 (получение)
  - `step_4_message` - Шаг 4 (отзыв)
  - `step_5_message` - Шаг 5 (реквизиты)
  - `order_pending_message` - Ожидание модерации
  - `payment_sent_message` - Выплата произведена

#### 6. **Улучшенная аналитика**
- **Лист2 (Аналитика)**: Обновляется в реальном времени
- **Лист1 (Заявки)**: Только завершенные заказы
- Защита от дублирования данных

## 📊 Изменения в базе данных

### Новые таблицы

#### `bot_config`
```sql
- id (INT)
- config_key (VARCHAR) UNIQUE
- config_value (TEXT)
- description (VARCHAR)
- updated_at (TIMESTAMP)
```

### Измененные таблицы

#### `products`
**Удалено:**
- `url` (теперь в инструкции)
- `cashback_amount` (теперь в инструкции)

**Осталось:**
- `id` (1-4)
- `name`
- `instruction_text`
- `is_active`

#### `orders`
**Добавлено:**
- `rejection_reason` (TEXT) - Причина отклонения

**Новые статусы:**
- `REVIEW_SENT` - Отзыв отправлен
- `PENDING_APPROVAL` - Ожидает одобрения админа
- `ADMIN_PAYMENT_SENT` - Админ отправил скрин перевода
- `REJECTED` - Отклонен

## 🔧 Технические изменения

### Backend
- **FastAPI** интеграция для Web App
- **Валидация Telegram initData** (HMAC-SHA256)
- **Async эндпоинты** для управления товарами и конфигом

### Frontend
- **Vanilla JS** (без фреймворков)
- **Telegram WebApp SDK**
- **Адаптивная верстка** (CSS Variables)

### Handlers
- Новые файлы:
  - `bot/handlers/client_new.py` - Пошаговая воронка
  - `bot/handlers/admin_new.py` - Упрощенная админка
- **FSM States** для управления состояниями

---

## ⚠️ Breaking Changes

1. **Товары нужно создавать заново** (структура изменилась)
2. **Старые заказы несовместимы** с новой схемой
3. **Требуется миграция базы** (или полная пересборка)
4. **Новая переменная** в `.env`: `WEBAPP_URL`
