# 🛒 Telegram Buyout Bot

Telegram бот для автоматизации выкупов товаров с WildBerries с кэшбэком. Полная воронка клиента, аналитика, интеграция с Google Sheets, админ-панель.

## 🎯 Функционал

### Клиентская часть
- ✅ Выбор товара из 4 слотов (с возможностью отключения)
- ✅ Пошаговая инструкция для каждого товара
- ✅ FSM-контроль загрузки скриншотов (корзина → покупка → получение → отзыв)
- ✅ Модерация финального отзыва админом
- ✅ Запрос реквизитов для кэшбэка
- ✅ Система поддержки (прямая связь с модераторами)

### Админ-панель
- ✅ Управление товарами (название, ссылка, сумма кэшбэка, активация/деактивация)
- ✅ Редактирование инструкций для каждого товара
- ✅ Модерация отзывов (одобрение/отклонение)
- ✅ Просмотр статистики воронки

### Интеграция с Google Sheets
- ✅ **Лист 1:** Автоматическое заполнение данных заказа при отправке скриншота покупки
- ✅ **Лист 2:** Аналитика конверсий по 7 этапам воронки (обновление каждые 5 минут)

## 🚀 Быстрый старт

### 1. Клонирование репозитория
```bash
git clone https://github.com/shogeo/telegram-buyout-bot.git
cd telegram-buyout-bot
```

### 2. Настройка окружения
Скопируйте `.env.example` в `.env` и заполните переменные:
```bash
cp .env.example .env
nano .env
```

**Обязательные переменные:**
- `BOT_TOKEN` — токен бота (от @BotFather)
- `ADMIN_IDS` — ID администраторов через запятую (получить через @userinfobot)
- `DATABASE_URL` — `postgresql+asyncpg://user:pass@postgres:5432/buyout_bot`
- `REDIS_URL` — `redis://redis:6379/0`
- `GOOGLE_SHEET_ID` — ID таблицы Google Sheets
- `GOOGLE_SERVICE_ACCOUNT_FILE` — путь к JSON-ключу сервисного аккаунта

### 3. Получение Google Service Account
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте проект → Включите Google Sheets API
3. Создайте сервисный аккаунт → Скачайте JSON-ключ
4. Расшарьте таблицу на email сервисного аккаунта (с правами редактора)
5. Сохраните JSON как `google_credentials.json` в корне проекта

### 4. Запуск через Docker
```bash
docker-compose up -d --build
```

Проверка логов:
```bash
docker-compose logs -f bot
```

### 5. Инициализация базы данных
БД создается автоматически при первом запуске. Для создания миграций:
```bash
docker-compose exec bot alembic revision --autogenerate -m "Initial migration"
docker-compose exec bot alembic upgrade head
```

## 🎮 Использование

### Для пользователей
1. Перейти по ссылке бота
2. Нажать "Выбрать товар для выкупа"
3. Следовать инструкциям (загрузка скриншотов)
4. После модерации отзыва — отправить реквизиты
5. Получить кэшбэк

### Для админов
Команды:
- `/admin` — открыть админ-панель
- `/stats` — посмотреть статистику воронки

Действия:
- **Редактировать товар** → выбрать слот → изменить данные
- **Модерировать отзывы** → одобрить/отклонить с комментарием

## 🛠 Технологический стек

- **Язык:** Python 3.11+
- **Бот:** aiogram 3.x (асинхронный фреймворк)
- **Веб-сервер:** FastAPI (для вебхуков)
- **База данных:** PostgreSQL + SQLAlchemy (async ORM)
- **Кеширование:** Redis (для FSM и аналитики)
- **Интеграции:** gspread_asyncio (Google Sheets API)
- **Контейнеризация:** Docker + docker-compose

## 🐛 Troubleshooting

**Ошибка: "Cannot connect to database"**
- Проверьте `DATABASE_URL` в `.env`
- Убедитесь что контейнер `postgres` запущен: `docker-compose ps`

**Ошибка: "Google Sheets API error"**
- Проверьте что таблица расшарена на email сервисного аккаунта
- Убедитесь что API включен в Google Cloud Console

**Бот не отвечает:**
- Проверьте логи: `docker-compose logs -f bot`
- Убедитесь что токен бота корректен
